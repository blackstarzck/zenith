## 3. 4% 고점에서 왜 매도하지 않았는가?

### 3.1 문제 현상

SIGN은 매수 후 최대 4% 상승했으나, 봇은 **+2.1% 시점에서 전량 청산**하고 나머지 상승분 ~2%p를 포착하지 못했습니다.

### 3.2 원인 ①: 매도 스코어가 수익률 2%에서 이미 너무 쉽게 70점 도달

매도 시점(+2.1%)의 스코어를 역산하면:

```python
# 수익률 2.13% → 수익 스코어
수익 = (2.13 / 3.0) × 100 = 71.0점     # 3%에서 만점인데 2%에서 이미 71점

# RSI가 70 부근 → RSI 과매수 스코어
RSI↑ = (70 - 40) / 30 × 100 = 100.0점

# 가격이 BB 상단 근처 → BB 포지션 스코어
BB↑ ≈ 99.0점

# ADX ~22 → 추세 스코어
ADX = (22 - 20) / 20 × 100 = 10.0점

# 총점 = (100 + 99 + 71 + 10) / 4 = 70.0 → 임계치 70.0 도달!
```

**수익률이 겨우 2%인데 매도 스코어가 70을 넘었습니다.** 수익 스코어 공식이 `profit_pct / 3.0 × 100`이므로, 3%에서 만점(100)인데 2%에서 이미 71점이라 다른 요소와 합산 시 바로 임계치를 넘깁니다.

### 3.3 원인 ②: 1차 익절 → 2차 전량매도까지 불과 4분

**설계 의도**:
```
1차 익절(50%) → 트레일링 스탑 활성화 → 고점 추적하며 나머지 보유 → 고점-ATR×2.0 하락 시 전량 매도
```

**실제 발생한 일**:
```
1차 익절(50%) → 4분 후 매도 스코어가 또 70 초과 → 즉시 SELL_ALL → 트레일링 스탑 작동할 기회 없음
```

`evaluate_exit`의 로직:
```python
# engine.py 283행
signal_type = Signal.SELL_ALL if has_sold_half else Signal.SELL_HALF
```

1차 익절 후 `has_sold_half = True`가 되면, 다음에 스코어가 임계치를 넘는 순간 **무조건 SELL_ALL**. 트레일링 스탑보다 스코어 조건이 **먼저 발동**하면 트레일링은 영영 작동하지 않습니다.

### 3.4 원인 ③: 스코어 익절과 트레일링 스탑의 경쟁 구조

```
evaluate_exit 우선순위:
  1순위: 동적 손절 (하드 룰) ← 가격 < entry - ATR×2.5
  2순위: 트레일링 스탑 (하드 룰) ← 1차 후, 가격 < 고점 - ATR×2.0
  3순위: 스코어링 익절 ← 스코어 ≥ 70.0 → SELL_ALL
```

2순위(트레일링)와 3순위(스코어)가 **경쟁**합니다:
- 트레일링은 **"고점에서 충분히 하락해야"** 발동
- 스코어는 **"수익이 조금만 있고 RSI가 높으면"** 바로 발동

**스코어가 항상 먼저 이기기** 때문에, 트레일링 스탑은 실질적으로 **사문화된 로직**입니다.

## 5. 개선 제안: 트레일링 스탑이 실제로 작동하게 만들기

```python
if has_sold_half:
    # 스코어링 익절 로직 도달 전에 return HOLD
    return TradeSignal(signal=Signal.HOLD, ...)
    # → 1순위(동적 손절)와 2순위(트레일링 스탑)만 작동
```
