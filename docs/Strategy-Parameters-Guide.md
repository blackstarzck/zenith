# 전략 파라미터 가이드 (Strategy Parameters Guide)

## 1. 개요

Zenith의 전략 수정 모달(`StrategyEditModal.tsx`)은 크게 두 부분으로 나뉩니다. 상단 섹션은 기술 지표를 계산하는 방식을 결정하고, 하단 섹션은 계산된 지표를 바탕으로 실제 매수 여부를 결정하는 논리를 제어합니다. 이 가이드는 각 설정값이 봇의 동작에 어떤 영향을 주는지 설명합니다.

## 2. 파라미터 설정 (지표 계산 방식)

이 섹션의 설정값은 기술 지표의 모양과 수치 자체를 바꿉니다. 비유하자면 온도계의 단위를 섭씨에서 화씨로 바꾸거나, 눈금의 간격을 조정하는 것과 같습니다. 측정 도구 자체를 변경하는 단계입니다.

(참조: `src/config.py` 37-111행 `StrategyParams`)

### 볼린저 밴드

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| `bb_period` | 20 | 이동평균선을 계산하는 기간. 값이 클수록 밴드가 완만해진다. |
| `bb_std_dev` | 2.0 | 밴드의 폭을 결정하는 표준편차 배수. 값이 클수록 밴드가 넓어져 신호가 적게 발생한다. |

### RSI

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| `rsi_period` | 14 | 상대강도지수를 계산하는 기간. |
| `rsi_oversold` | 30 | 과매도 구간을 정의하는 기준점. |

### ATR 손절

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| `atr_period` | 14 | 변동성(ATR)을 계산하는 기간. |
| `atr_stop_multiplier` | 2.5 | ATR에 이 값을 곱해 손절 거리를 정한다. 값이 클수록 손절선이 멀어진다. |

## 3. 매수 진입 조건 (스코어링 시스템)

지표가 계산되면 봇은 6가지 항목에 대해 점수를 매긴다. 각 항목의 중요도인 가중치를 조절하여 봇의 성향을 바꿀 수 있다. 비유하자면 시험 과목별 배점과 합격 커트라인을 정하는 것과 같다.

(참조: `src/strategy/engine.py` 57-127행 `evaluate_entry()`)

### 스코어링 공식

`src/strategy/engine.py` 101행:

```
총점 = (w_volatility x 점수 + w_ma_trend x 점수 + w_adx x 점수 + w_bb_recovery x 점수 + w_rsi_slope x 점수 + w_rsi_level x 점수) / 가중치합
```

### 6가지 가중치 항목

(참조: `src/strategy/engine.py` 131-173행)

| 가중치 | 기본값 | 의미 | 점수 산정 기준 |
|--------|--------|------|--------------|
| `w_volatility` | 1.0 | 시장 안정성 | 변동성 비율 1.0 이하 = 100점, 3.0 이상 = 0점 |
| `w_ma_trend` | 1.0 | 이평선 추세 (골든크로스) | 20일선 > 50일선 = 100점, 그 외 = 0점 |
| `w_adx` | 1.0 | 추세 강도 (횡보 선호) | ADX 15 이하 = 100점, 40 이상 = 0점 |
| `w_bb_recovery` | 1.0 | 볼린저 밴드 복귀 | 하단 이탈 후 복귀 = 100점, 이탈 중 = 30점, 해당 없음 = 0점 |
| `w_rsi_slope` | 1.0 | RSI 상승 기울기 | 기울기 3.0 이상 = 100점, 0 이하 = 0점 |
| `w_rsi_level` | 1.0 | RSI 레벨 (과매도) | RSI 20 이하 = 100점, 45 이상 = 0점 |

가중치를 0으로 설정하면 해당 조건을 완전히 무시한다. 가중치가 높을수록 해당 조건이 총점에 미치는 영향이 커진다.

### 진입 스코어 임계값 (`entry_score_threshold`)

가중치로 계산된 총점이 이 값을 넘어야 매수 주문을 낸다.

| 임계값 | 성격 | 설명 |
|--------|------|------|
| 55 | 공격적 | 조금만 반등 기미가 보여도 매수. 거래 횟수 많고 승률 낮음. |
| **70** | **균형 (기본값)** | 여러 지표가 공통적으로 반등을 가리킬 때 진입. |
| 90 | 보수적 | 모든 조건이 완벽하게 맞아야 매수. 거래 횟수 적지만 확실한 자리만. |

## 4. 실전 예시

현재 시장 상황이 다음과 같다고 가정한다:

| 지표 | 현재 값 | 점수 |
|------|---------|------|
| RSI | 28 | 68점 |
| ADX | 18 | 88점 |
| 변동성 비율 | 1.2 | 90점 |
| BB 상태 | 하단 이탈 후 복귀 | 100점 |
| 이평선 | 20일선 > 50일선 | 100점 |
| RSI 기울기 | 2.1 | 70점 |

### 기본 가중치(모두 1.0) 적용 시

```
총점 = (1.0x90 + 1.0x100 + 1.0x88 + 1.0x100 + 1.0x70 + 1.0x68) / 6.0
     = 516 / 6
     = 86.0점
```

결과: 임계값 70점을 넘었으므로 **매수(BUY)** 신호를 생성한다.

### 설정을 바꾸면 어떻게 달라지는가?

**파라미터 변경 (상단 섹션)**: `rsi_period`를 14에서 7로 줄이면 RSI 수치 자체가 28에서 다른 값으로 변한다. 측정 도구가 바뀌므로 모든 계산의 시작점이 달라진다.

**진입 조건 변경 (하단 섹션)**: `w_rsi_level`을 1.0에서 3.0으로 올리면 RSI 28이라는 수치는 그대로이나, 전체 점수에서 RSI 레벨이 차지하는 비중이 3배가 된다. RSI가 조금만 높아도 총점이 크게 깎인다.

## 5. 한눈에 비교

| 구분 | 파라미터 설정 | 매수 진입 조건 |
|------|-------------|--------------|
| 바꾸는 것 | 지표 계산 공식의 입력값 | 지표를 보는 눈높이와 기준점 |
| 영향 | BB 모양, RSI 수치, 손절 거리 | 같은 시장 상황에서 매수 여부 결정 |
| 비유 | 온도계 눈금 조정 | 에어컨 설정 온도 변경 |
| 코드 위치 | `indicators.py`에서 계산 시 사용 | `engine.py`의 `evaluate_entry()`에서 사용 |

| 임계값 | 거래 수 | 승률 | 수익률 | 비고 |
|--------|---------|------|--------|------|
| 85 | 0건 | - | - | 너무 엄격하여 기회를 모두 놓침 |
| **70** | **14건** | **50.0%** | **+0.13%** | **안정적 흐름, 채택** |
| 55 | 49건 | 46.2% | -0.76% | 잦은 매매로 수수료 손실 발생 |
