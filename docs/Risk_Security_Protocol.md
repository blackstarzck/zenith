# 리스크 관리 및 보안 프로토콜 (Risk & Security)

## 1. 자산 보호 전략 (Risk Management)
- **최대 손실 제한(Daily Stop-loss):** 하루 전체 자산의 5% 이상 손실 발생 시 당일 모든 매매 즉시 중단.
- **종목별 비중 제한:** 한 종목이 전체 자산에서 차지하는 비중을 최대 20%로 제한하여 특정 코인의 급락에 대비.
- **미체결 주문 관리:** 주문 후 5분 내 체결되지 않은 주문은 자동 취소하여 자금의 잠김 방지.

## 2. 보안 가이드라인 (Security)
- **API Key 관리:** 업비트 및 Supabase API Key는 소스 코드에 직접 입력하지 않고, 환경 변수(`.env`) 파일에 저장하여 외부 유출 방지.
- **최소 권한 원칙:** 업비트 API 발급 시 '출금' 권한은 반드시 제외하고 '조회'와 '주문' 권한만 부여.
- **로컬 접근 제한:** 개인 PC에서 구동되므로, 외부로부터의 원격 접속(SSH 등)을 차단하거나 보안 설정 강화.

## 3. 비상 대응 절차 (Emergency Plan)
- **네트워크 단절:** 인터넷 연결이 끊길 경우, 프로그램을 재시작할 때 미체결 주문과 현재 보유 잔고를 우선 확인하는 복구 로직 실행.
- **API 오류:** 업비트 서버 점검 또는 API 오류 발생 시 카카오톡으로 긴급 알림 전송 후 대기 모드 진입.
