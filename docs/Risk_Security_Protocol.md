# 리스크 관리 및 보안 프로토콜 (Risk & Security)

## 1. 자산 보호 전략 (Risk Management)
- **최대 손실 제한(Daily Stop-loss):** 하루 전체 자산의 5% 이상 손실 발생 시 당일 모든 매매 즉시 중단.
- **종목별 비중 제한:** 한 종목이 전체 자산에서 차지하는 비중을 최대 20%로 제한하여 특정 코인의 급락에 대비.
- **미체결 주문 관리:** 주문 후 30초 내 체결되지 않은 주문은 자동 취소하여 자금의 잠김 방지.

### 1.1. 켈리 공식 기반 포지션 사이징 (Kelly Criterion)
- **Half-Kelly 적용:** 과거 승률(p)과 손익비(b)를 기반으로 최적의 투자 비중(f*)을 산출한 후, 보수적 운용을 위해 0.5를 곱한 Half-Kelly 비중을 사용. (공식: f_applied = (p - (1-p)/b) × 0.5)
- **최소 거래수 조건:** 신뢰도 확보를 위해 매도 완료된 거래가 30회 미만인 경우, 기존의 고정 비중(20%)을 기본값으로 사용.
- **최대 비중 캡:** 켈리 공식 결과가 높더라도 전체 자산의 최대 20%를 초과할 수 없음.
- **진입 차단:** 기대값이 음수(Kelly ≤ 0)인 경우 해당 시점의 진입을 원천 차단.
### 1.2. 레짐 적응형 ATR 손절 (Regime-Adaptive Stop Loss)
- **동적 배수 적용:** 시장 레짐(횡보/추세/변동성)에 따라 ATR 손절 배수를 차등 적용하여 레짐별 최적 방어선을 유지.
  - 횡보장(ranging): ATR × 2.8 — 노이즈 허용 여유
  - 추세장(trending): ATR × 2.2 — 역추세 빠른 탈출
  - 변동성 폭발(volatile): ATR × 2.5 — 중립
  - 미분류: ATR × 3.0 — 안전 폴백
- **단방향 레짐 홀드:** 안전 모드(추세/변동성) 진입은 즉시 허용하되, 해제(→횡보)는 최소 20분 유지하여 레짐 플래핑 방지.
- **잔량 이익 잠금:** 1차 익절 후에는 트레일링 스탑 기본 배수 2.4를 사용하고, 레짐/수익률에 따라 배수를 자동 보정합니다.
- **본전+마진 보호:** 트레일링 스탑이 과도하게 느슨해지지 않도록 `entry_price × (1 + min_profit_margin)` 바닥가를 적용합니다.

### 1.3. 연속 손절 브레이커 (Consecutive Stop-Loss Breaker)
- **연쇄 손실 방지:** 30분 내 2건 이상 손절 발생 시 신규 매수를 30분간 차단.
- **기존 포지션 영향 없음:** 이미 보유 중인 포지션의 손절/익절은 정상 작동하며, 신규 진입만 차단.
- **자동 해제:** 차단 시간(30분) 경과 후 자동으로 매수 허용 복구.


### 1.4. 슬리피지 제어 (Slippage Control)
- **호가창 기반 시뮬레이션:** 매수 주문 전 호가창(Orderbook)을 분석하여 주문 금액에 따른 예상 체결 가격을 계산(Walk-the-book).
- **진입 제한:** 예상 슬리피지가 50bps(0.5%)를 초과할 경우, 시장가 매수로 인한 과도한 비용 발생을 방지하기 위해 진입을 거부.

### 1.5. 익절 품질 게이트 (Take-Profit Quality Gate)
- **핵심 목적:** 조기 익절을 줄이고 손익비를 높여 안정적 수익을 확보.
- **적용 조건:** 1차 익절(기본 40%)은 아래 2단계 통과 시에만 실행.
  - 적응형 최소 수익률: `profit_pct ≥ max(min_profit_margin, ATR/entry × 0.8, 상한 1.2%)`
  - 품질 검증: `가격 ≥ BB 중앙선` 또는 `profit_pct ≥ 적응형 최소수익률 × 1.8`
- **설계 이유:** 스코어 단독 청산은 노이즈 구간에서 너무 빨리 이익을 확정하는 경향이 있어, 평균복귀 확인과 변동성 반영 마진을 결합해 기대값을 개선합니다.
- **BB 상단 하드 룰 우회:** 가격이 BB 상단 이상이고 최소 수익 마진(0.45%)을 충족하면, 위 품질 게이트를 거치지 않고 즉시 매도합니다. 이 하드 룰은 스코어링 임계치·적응형 마진·품질 검증을 모두 우회하여, 확실한 익절 기회를 놓치지 않도록 설계되었습니다.
## 2. 보안 가이드라인 (Security)
- **API Key 관리:** 업비트 및 Supabase API Key는 소스 코드에 직접 입력하지 않고, 환경 변수(`.env`) 파일에 저장하여 외부 유출 방지.
- **최소 권한 원칙:** 업비트 API 발급 시 '출금' 권한은 반드시 제외하고 '조회'와 '주문' 권한만 부여.
- **로컬 접근 제한:** 개인 PC에서 구동되므로, 외부로부터의 원격 접속(SSH 등)을 차단하거나 보안 설정 강화.

## 3. 비상 대응 절차 (Emergency Plan)
- **네트워크 단절:** 인터넷 연결이 끊길 경우, 프로그램을 재시작할 때 미체결 주문과 현재 보유 잔고를 우선 확인하는 복구 로직 실행.
- **API 오류:** 업비트 서버 점검 또는 API 오류 발생 시 카카오톡으로 긴급 알림 전송 후 대기 모드 진입.
