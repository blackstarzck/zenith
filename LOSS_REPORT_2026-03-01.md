## 4. 손실 원인 분석

### 4.1 전략 레벨 문제

#### 🔴 진입 스코어 임계치가 하락장에서 너무 관대함
- 횡보장 기준 임계치 **70.0**이 적용되었으나, 손절된 4건 모두 진입 스코어가 **71~76점** 범위로 — 겨우 통과한 수준
- 만약 `regime_trending_offset` (+15.0)이 적시에 적용되었다면 **임계치 85.0**으로 4건 모두 **진입 차단**되었을 것

#### 🔴 BB Recovery 시그널의 한계 — "일시적 반등 vs 추세 반전" 미구분
- 하락 추세에서 볼린저 밴드 하단을 이탈한 후 **일시적으로 밴드 내로 복귀**하면 `recovered` (100점) 부여
- 그러나 오늘처럼 추세적 하락에서의 복귀는 **데드캣 바운스(Dead Cat Bounce)**에 불과
- BB Recovery가 100점을 부여하면서 총점을 70 이상으로 끌어올리는 **주된 점수 부풀리기 요인**

#### 🔴 RSI 극저값이 "떨어지는 칼날 잡기"를 유도
- 손절된 종목들의 진입 시 RSI 레벨 스코어: **0~13점** (RSI 45 이상이면 0점)
- 이 스코어 자체는 낮지만, RSI가 극단적으로 낮다는 것은 매도 압력이 극심하다는 의미
- RSI 기울기(RSI↗) 스코어가 **63~100점**으로 높아 단기 반등을 감지했으나, 이 반등이 지속되지 않음

#### 🟡 시장 레짐 감지의 후행성
현재 레짐 감지 로직:
```
# regime.py — BTC-KRW 기반 레짐 분류
ADX ≥ 25.0 → Trending (offset +15.0)
Vol Ratio ≥ 2.0 → Volatile (offset +25.0)
기본값 → Ranging (offset 0)
히스테리시스: 3캔들 다수결
```
- `regime_lookback_candles = 3` (히스테리시스)이 전환 속도를 늦춤
- **10분 간격** 갱신 주기 + **15분봉 3캔들 다수결** = 최소 **55분 이상** 지연 가능
- 07:00 하락 시작 → 10:19 Trending 감지 = **3시간 19분 지연**

### 4.2 리스크 관리 문제

#### 🟡 ATR×2.5 손절선이 알트코인 변동성 대비 타이트함
- 현재 설정: `atr_stop_multiplier = 2.5` (진입가 - ATR×2.5 = 손절선)
- 15분봉 기준 ATR 대비 알트코인의 **일중 변동폭이 4~5%**에 달해, 정상적인 가격 노이즈에도 손절선 터치 빈번
- 특히 XRP(-828), SOL(-978)은 손절선과 체결가의 차이가 **1원 이내**로 매우 근접 — 약간의 여유만 있었어도 반등 포착 가능

#### 🟡 Kelly Fraction 미작동
- `bot_state.kelly_fraction`이 `null`로 기록 — **Half-Kelly 동적 포지션 사이징 미작동**
- 원인: `kelly_min_trades = 30` (최소 샘플 수) 미달로 고정 비율(20%) 폴백
- 연패 구간에서 포지션 크기가 줄어들지 않아 손실 규모가 일정하게 유지

#### 🟢 일일 손실 한도
- 초기 자본 ~250,315 KRW 대비 손실 -1,559 KRW = **약 0.6%** → 한도 5% 미도달
- 일일 손실 한도 시스템은 **정상 작동** 중 (다행히 발동되지 않음)


### 5.2 문제의 핵심

**BB Recovery(100점) + Vol(100점) + MA(100점)**의 3개 컴포넌트가 만점을 기록하면서, RSI 수준이 극히 낮아도(0~13점) 총점이 70점을 초과했습니다.

6개 컴포넌트 중 **3개가 상시 만점 경향**을 보이면:
- 최소 보장 점수 = (100+100+100+0+0+0)/6 = **50점**
- BB(30) + RSI↗(50)만 추가되어도 = (100+100+100+30+50+0)/6 = **63.3점**
- BB(100) + RSI↗(63)이면 = (100+100+100+100+63+0)/6 = **77.2점** → 70 초과

→ **실질적으로 BB Recovery가 "recovered"이기만 하면 거의 무조건 진입**

### 6.2 로직 개선 (코드 수정 필요)

#### ① RSI 극저값 필터 (Falling Knife Guard)
```
RSI < 15 → BB Recovery 스코어를 100 대신 30으로 제한
(극단적 과매도에서의 반등 신호 신뢰도를 낮춤)
```

#### ② BB Recovery 추세 확인 보강
```
BB Recovery "recovered" 판정 시 추가 조건:
- MA20이 하락 중이면 recovered → 30점으로 하향 (추세적 하락 중 일시 반등)
- MA20이 상승 또는 횡보이면 recovered → 100점 유지 (진정한 반등)
```

#### ③ 레짐 감지 주기 단축
```
현재: 10분마다 갱신 (loop_count % 60 == 1)
제안: 2분마다 갱신 (loop_count % 12 == 1)
→ 레짐 전환 최대 지연 = 2분 + 1~2캔들 히스테리시스 ≈ 17~32분
```

#### ④ 연속 손절 브레이커
```
최근 30분 내 2건 이상 손절 발생 시:
→ 신규 매수 30분 차단 (전략 조건 무관)
→ 연쇄 손절 방지 및 시장 안정화 대기
```

### 6.3 중기 개선 (아키텍처)

- **추세 추종 모드 추가**: 레짐이 Trending일 때 평균회귀 대신 모멘텀/브레이크아웃 전략으로 전환하는 하이브리드 시스템
