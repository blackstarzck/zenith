## Task 3 — evaluate_entry() Scoring Rewrite
- Scoring formula: `total_score = sum(w * s for w, s in scores.values()) / total_weight`
- 6 scoring methods: _score_volatility, _score_ma_trend, _score_adx, _score_bb_recovery, _score_rsi_slope, _score_rsi_level
- All use linear interpolation clamped to [0, 100]
- `calc_bb_status()` replaces stateful `_was_below_lower` dict — fully stateless now
- `recover_bb_state()` and `reset_tracking()` removed from engine.py (no more stateful BB tracking)
- External call sites removed: orchestrator.py (3x reset_tracking, 1x recover_bb_state call, _recover_bb_states method, __init__ call), paper_trading.py (1x reset_tracking)
- QA test with threshold=60.0 returns Score=46.7 (HOLD) — BB:0 and RSI↗:0 drag it down despite Vol:100, ADX:100, RSI:80
- All-zero weights correctly returns HOLD with score=0.0
- MARKET_PAUSE signal no longer generated by evaluate_entry() (was volatility gate, now scored)
- Orchestrator still checks for Signal.MARKET_PAUSE — kept as dead code for safety per task spec
- 진입 알고리즘을 기존 AND-gate 필터에서 가중치 기반 스코어링 시스템으로 전환함.
- 6가지 필터(변동성, MA 추세, ADX, BB 복귀, RSI 기울기, RSI 수준)에 대해 각각 0~100점의 점수를 산출함.
- 각 필터별 가중치(0~10)를 설정할 수 있으며, 가중 평균 점수가 임계치(기본 85.0) 이상일 때 진입함.
- BB 복귀 상태는 를 통해 무상태(stateless)로 계산되도록 변경됨.
- 프론트엔드 에 가중치 조절 슬라이더와 임계치 입력 필드가 추가됨.
- 진입 알고리즘을 기존 AND-gate 필터에서 가중치 기반 스코어링 시스템으로 전환함.
- 6가지 필터(변동성, MA 추세, ADX, BB 복귀, RSI 기울기, RSI 수준)에 대해 각각 0~100점의 점수를 산출함.
- 각 필터별 가중치(0~10)를 설정할 수 있으며, 가중 평균 점수가 임계치(기본 85.0) 이상일 때 진입함.
- BB 복귀 상태는 calc_bb_status()를 통해 무상태(stateless)로 계산되도록 변경됨.
- 프론트엔드 StrategyEditModal에 가중치 조절 슬라이더와 임계치 입력 필드가 추가됨.
